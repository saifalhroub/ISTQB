# Section 1 — Chapter 1 (Fundamentals of Testing)

## Q1
Which of the following BEST describes the main purpose of testing?

A. To prove the software has no defects  
B. To increase the quality directly  
C. To measure quality and reduce risk  
D. To ensure developers wrote correct code

- Answer: C
---

## Q2
Which of the following are VALID objectives of testing?

(Choose TWO)

A. Prevent defects  
B. Prove the system is error-free  
C. Build confidence in the system  
D. Guarantee that no failures will occur

- Answer: A & C

---

## Q3
Which of the following activities helps **prevent defects early**?

A. Running regression tests  
B. Reviewing requirements and documents  
C. Executing test cases  
D. Fixing production bugs

- Answer: B

---

## Q4
Which statement about **testing and quality** is TRUE?

A. Testing improves quality by itself  
B. Testing measures quality and helps identify defects  
C. Quality can only be improved by testing  
D. Testing guarantees defect-free software

- Answer: B

---

## Q5
Which of the following are **high-level test activities**?

(Choose TWO)

A. Test Planning  
B. Test Control  
C. Writing detailed test cases  
D. Executing test scripts

- Answer: A & B

---

## Q6
Which of the following is the CORRECT order of test process activities?

A. Planning → Execution → Analysis → Reporting  
B. Planning & Control → Analysis & Design → Implementation & Execution → Exit Criteria & Reporting  
C. Analysis → Planning → Execution → Closure  
D. Implementation → Planning → Execution → Reporting

- Answer: B

---

## Q7
What is the difference between **functional testing** and **non-functional testing**?

A. Functional tests internal structure while non-functional tests user interface  
B. Functional tests what the system does, non-functional tests how well it performs  
C. Functional testing is only done by developers  
D. Non-functional testing only happens after release

- Answer: B

---

## Q8
Which of the following is an example of **non-functional testing**?

A. Login functionality  
B. Shopping cart calculation  
C. System performance under heavy load  
D. Password validation rules

- Answer: C

---

# Section 4 — Scenario Questions

## Q25

A development team integrates all modules at once and begins testing the entire system.  
They struggle to identify which module caused failures.

Which integration strategy are they using?

A. Top Down  
B. Bottom Up  
C. Big Bang  
D. Incremental

- Answer: C

---

## Q26

A tester reviews requirement documents before development starts and identifies missing conditions.

What testing activity is this?

A. Dynamic Testing  
B. Static Review  
C. Regression Testing  
D. System Testing

- Answer: B

---

## Q27

A system modification was implemented to add a new payment method.  
The testing team runs old test cases to ensure existing features still work.

What type of testing is this?

A. Confirmation Testing  
B. Regression Testing  
C. Acceptance Testing  
D. Exploratory Testing

- Answer: B

# Section 5 — True / False

## Q28
Testing can prove that the software contains no defects.

True / False

- Answer: False

---

## Q29
Static testing detects defects but not failures.

True / False

- Answer: True

---

## Q30
Regression testing ensures that changes did not negatively impact existing functionality.

True / False

- Answer: True
